# Emergent Engine — 장기 개인화 AI 협업 플랫폼

> "두 AI가 서로 검증하며 성장하는 개인 AI 두뇌"

---

## 핵심 차별점: OpenClaw와의 비교

| | OpenClaw | Emergent Engine |
|--|---------|-----------------|
| AI 수 | 1개 (록이) | 2개 (A+B 협업) |
| 기억 방식 | MEMORY.md (파일) | KG (구조화 그래프) |
| 검증 | 없음 | B가 A를 항상 검증 |
| 개인화 속도 | 빠름 | 느리지만 더 깊음 |
| 오류율 | 보통 | 낮음 (상호검증) |

**핵심**: OpenClaw가 "1인 AI 비서"라면, Emergent Engine은 "AI 2인팀 비서"

---

## 아키텍처

```
사용자 질문
    ↓
[협업 미들웨어]
  ├─ Agent A: 답변 생성 (전문가)
  ├─ Agent B: 독립 검증 (비판자)
  ├─ 합의 엔진: 두 답변 조율
  └─ KG 업데이트: 이 대화를 지식으로 저장
    ↓
검증된 답변 + KG 누적
```

## Cold Start 문제 해결책

**3단계 부트스트래핑:**

1. **Day 1**: 범용 기반 KG (도메인별 사전 구축)
   - 투자: 주요 지표 정의, 산업 분류
   - 코드: 언어/프레임워크 지식
   - 일반: 상식 레이어

2. **Week 1**: 사용자 패턴 학습
   - 어떤 질문을 자주 하나?
   - 어떤 오류를 반복하나?
   - 선호 표현 방식

3. **Month 1+**: 진정한 개인화
   - "상록은 항상 배당주에 관심"
   - "기술 분석보다 펀더멘털 선호"
   - → 상록만을 위한 KG 완성

## 실용 제품 3개 (MVP 우선순위)

### 1. 🥇 투자 분석 보조 (프리즘 인사이트 연계)

```
사용자: "삼성전자 분석해줘"

Solo GPT:
→ 일반적인 분석 (사용자가 맞는지 모름)

Emergent Engine:
A: "매출 성장률 둔화, PER 15x로 적정"
B: "반박: 반도체 사이클 고려 안 함. PER 재산정 필요"
합의: "PER 15x는 사이클 정상화 가정시 적정, 
      하지만 2025Q3 실적 확인 필요 — 불확실성 명시"
KG: 이 분석 패턴 저장
```

**가치**: 틀린 분석을 자동 검증. 상록 채널 독자들에게 신뢰성 ↑

### 2. 🥈 코드 리뷰 봇 (개발팀)

```
Solo: "코드 LGTM"  ← 버그 놓침
Emergent: A가 리뷰 → B가 독립 재검토 → 놓친 버그 발견
```

**가치**: Solo AI 리뷰보다 오류 감지율 높음 (실험 증명 예정)

### 3. 🥉 개인 AI 비서 (OpenClaw 다음 버전)

```
KG에 쌓인 개인 맥락:
- 상록의 투자 철학
- 선호 정보 형식
- 반복 질문 패턴

→ 6개월 후: "상록 전용 AI"가 됨
```

## 실험으로 증명된 것 (오늘 기준)

| 실험 | Solo | 협업 | 차이 |
|------|------|------|------|
| 새 연산 규칙(vex) | 20% | 100% | **+80%p** |
| 논리 추론(knights) | 70% | 60~80% | 복잡 |
| 수학 계산(aime) | 50% | 75% | +25%p |

**핵심 메시지**: 새롭고 구조화된 문제에서 협업이 Solo를 크게 앞섬

## 논문 새 프레임 (오늘 합의)

**기존**: "창발 vs 비창발 비교" (GPT-5.2 reject)

**새안**: "동일 모델 기반 협업 구조별 성능 비교"
- Solo (1-call): 기준선
- Pipeline (4-call, 단계적): 표준 multi-agent
- Emergent (4-call, 적대적): 우리 방식
- **새 규칙 도메인에서 Emergent ≥ Pipeline >> Solo**

이건 GPT-5.2도 반박하기 어려운 empirical claim
