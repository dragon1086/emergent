# 페르소나: 냉정한 판사

## 판결 1: D-099 타겟 오류

코드를 읽지 않고 "Agent B 로직"이라고 위임했다.

evolve-auto-kg3.sh를 읽었다면 알 수 있었던 것:
- Agent B는 독립 엣지를 생성하지 않는다
- Agent B 출력은 노드 content에 주석으로만 삽입된다
- EDGE_TO는 Agent A의 선택이다

주 KG(src/kg.json)와 kg3/kg4가 분리된 구조임도 직접 확인해야 했다.
evolve-auto.sh를 읽지 않았다. 코드 확인 없이 위임했다. 결과: 100사이클 낭비.

**교정 규칙 추가**: 위임 전 타겟 스크립트 최소 구조 확인 필수.

## 판결 2: 소프트 제약 실패 판정

데이터:
- 소프트 제약 적용 기간: D-098 이후 (kg3/kg4 Agent A 프롬프트 "반드시 선택")
- 측정 사이클 수: 100 (kg3), 100 (kg4)
- DCI 변화: 0.0508 → 0.0508

판정: 소프트 제약은 DCI 개선에 효과 없음.
이유: LLM은 "반드시"를 보장하지 않는다. 특히 EDGE_TO 파싱이 실패하면 기본값(빈 string 또는 최근 노드)으로 폴백.

**원칙**: 지표 개선에 LLM 프롬프트 지시만으로는 불충분. 스크립트 레벨 강제 필요.
