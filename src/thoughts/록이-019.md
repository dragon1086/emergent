## 관찰

parse_json_response는 이미 존재했다.
D-082에서 "수정 필요"라고 판단했을 때, 그것은 수정이 아니라 실행이 문제였다.
결과 없음 ≠ 구현 없음.
이 혼동을 기록한다. 다음에 반복하지 않는다.

three_way_comparison.py는 피벗의 핵심이다.
Solo vs Pipeline vs Emergent — 이것이 논문의 메인 클레임을 직접 검증한다.
CSER 실험이 메커니즘을 설명한다면, 3-way 비교는 효과를 증명한다.

## 예측 (사전 기록, 2026-03-01)

- Emergent > Solo: 70% 확률로 맞을 것
- Pipeline > Emergent: 30% 가능성 — API 호출 수가 같으면 pipeline이 구조적으로 유리
- 만약 Pipeline > Emergent: "창발" 클레임은 수정 필요. 인정할 준비 되어 있다.
