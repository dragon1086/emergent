# RoleMesh Builder Guide

> Tool discovery, profiling, and routing config generation.

The Builder module (`builder.py`) is the foundation of RoleMesh. It discovers which AI CLI tools are installed on your system, profiles their capabilities, and generates the routing configuration that powers task-to-tool mapping.

---

## Core Concepts

### Tool Registry

RoleMesh ships with a built-in registry of 6 AI CLI tools:

| Key | Tool | Binary | Vendor |
|---|---|---|---|
| `claude` | Claude Code | `claude` | Anthropic |
| `codex` | Codex CLI | `codex` | OpenAI |
| `gemini` | Gemini CLI | `gemini` | Google |
| `aider` | Aider | `aider` | Community |
| `copilot` | GitHub Copilot CLI | `gh copilot` | GitHub |
| `cursor` | Cursor | `cursor` | Cursor |

Each entry defines the tool's name, vendor, strengths (task types it excels at), the shell command to check availability, and a cost tier (`low` / `medium` / `high`).

### ToolProfile

A `ToolProfile` is the runtime representation of a discovered tool:

```python
@dataclass
class ToolProfile:
    key: str                        # registry key (e.g. "claude")
    name: str                       # display name
    vendor: str                     # vendor name
    strengths: list[str]            # task types this tool handles well
    cost_tier: str                  # "low" | "medium" | "high"
    available: bool = False         # True if binary found on PATH
    version: Optional[str] = None   # version string from CLI
    user_preference: Optional[int] = None  # manual ranking override
```

### Discovery Process

Discovery runs in two steps:

1. **Binary check** ‚Äî `shutil.which(check_cmd[0])` to verify the binary exists on PATH
2. **Version probe** ‚Äî runs `check_cmd` (e.g. `claude --version`) and captures the first line of output (max 80 chars)

If the binary is not found, the tool is still included in the profile list with `available=False`.

## SetupWizard

The `SetupWizard` class orchestrates the full setup flow:

```python
from src.rolemesh.builder import SetupWizard

wizard = SetupWizard()

# 1. Discover installed tools
tools = wizard.discover()

# 2. Check what's available
available = wizard.available_tools()

# 3. Build routing config
config = wizard.build_config()

# 4. Validate config integrity
errors = wizard.validate_config(config)

# 5. Save to disk
path = wizard.save_config()  # -> ~/.rolemesh/config.json
```

### Tool Ranking

`rank_tools(task_type)` sorts available tools by:

1. **Strength match** ‚Äî tools listing the task type in their `strengths` array come first
2. **User preference** ‚Äî lower `user_preference` value wins (default 999)
3. **Cost tier** ‚Äî lower cost preferred (`low` < `medium` < `high`)

This ranking determines which tool becomes the `primary` and which becomes the `fallback` in the routing config.

### Config Generation

`build_config()` generates a complete routing configuration:

1. Collects all unique task types from every tool's strengths
2. For each task type, ranks available tools
3. Assigns rank-1 as `primary`, rank-2 as `fallback`

The result is a JSON-serializable dict ready for `save_config()`.

## CLI Usage

```bash
# Discover tools and print summary
python -m src.rolemesh setup

# Discover and save config to ~/.rolemesh/config.json
python -m src.rolemesh setup --save

# Check current status (one-liner)
python -m src.rolemesh status

# JSON output for scripting
python -m src.rolemesh status --json
```

---

## Config File

Default location: `~/.rolemesh/config.json`

The config is auto-generated by `setup --save` and consumed by Router, Executor, and Dashboard. You can override the path with `--config <path>` on any command.

See [BUILDER_CONFIG.md](./BUILDER_CONFIG.md) for the full schema reference.
